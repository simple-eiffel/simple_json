Overview

SIMPLE_JSON is the main entry point for all JSON operations in the library. It provides a simple, high-level API for parsing JSON text and building JSON values with full Unicode/UTF-8 support.

API Reference

Parsing Operations

parse (a_json_text: STRING_32): detachable SIMPLE_JSON_VALUE

Parse JSON text and return a SIMPLE_JSON_VALUE wrapper.

local json: SIMPLE_JSON value: detachable SIMPLE_JSON_VALUE do create json value := json.parse ('{"name": "Alice", "age": 30}') if attached value then -- Success: value contains parsed JSON else -- Error: check json.last_errors for details print (json.errors_as_string) end end

Preconditions:

  • not_empty: Input text must not be empty

Returns: SIMPLE_JSON_VALUE on success, Void on error

Error Handling: On error, returns Void and populates last_errors with detailed information

parse_file (a_file_path: STRING_32): detachable SIMPLE_JSON_VALUE

Parse JSON from a file with automatic UTF-8 conversion.

local json: SIMPLE_JSON value: detachable SIMPLE_JSON_VALUE do create json value := json.parse_file ("data.json") if attached value then -- File parsed successfully else if json.has_errors then across json.last_errors as ic loop print (ic.to_string_with_position) print ("%N") end end end end

Preconditions:

  • not_empty: File path must not be empty

File Handling:

  • Automatically checks file existence and readability
  • Converts UTF-8 file content to STRING_32
  • Adds file access errors to last_errors if file cannot be read

is_valid_json (a_json_text: STRING_32): BOOLEAN

Check if text is valid JSON without creating the value object.

local json: SIMPLE_JSON do create json if json.is_valid_json ('{"test": true}') then print ("Valid JSON") else print ("Invalid JSON: " + json.errors_as_string) end end

Use Case: Validate JSON before parsing when you only need to know if it's valid

Performance: More efficient than parse when you don't need the parsed value

Building Operations

new_object: SIMPLE_JSON_OBJECT

Create a new JSON object builder for fluent construction.

local json: SIMPLE_JSON obj: SIMPLE_JSON_OBJECT do create json obj := json.new_object .put_string ("Alice", "name") .put_integer (30, "age") .put_boolean (True, "active") print (obj.to_json_string) -- Output: {"name":"Alice","age":30,"active":true} end

new_array: SIMPLE_JSON_ARRAY

Create a new JSON array builder for fluent construction.

local json: SIMPLE_JSON arr: SIMPLE_JSON_ARRAY do create json arr := json.new_array .add_string ("red") .add_string ("green") .add_string ("blue") print (arr.to_json_string) -- Output: ["red","green","blue"] end

Error Tracking

has_errors: BOOLEAN

Check if errors occurred during the last operation.

last_errors: ARRAYED_LIST [SIMPLE_JSON_ERROR]

Get list of all errors from last operation with full details.

error_count: INTEGER

Number of errors from last operation.

first_error: detachable SIMPLE_JSON_ERROR

Get first error if any exist (convenience method).

errors_as_string: STRING_32

All errors formatted as a single string with position information.

detailed_errors: STRING_32

All errors with detailed position information and source context.

clear_errors

Manually clear all error information (automatically done before each parse).

local json: SIMPLE_JSON value: detachable SIMPLE_JSON_VALUE do create json value := json.parse ('{invalid json}') if json.has_errors then print ("Error count: " + json.error_count.out + "%N") print ("Errors:%N" + json.errors_as_string + "%N") print ("%NDetailed:%N" + json.detailed_errors) end end

Complete Examples

Example 1: Basic Parsing and Type Checking

local json: SIMPLE_JSON value: detachable SIMPLE_JSON_VALUE obj: SIMPLE_JSON_OBJECT do create json value := json.parse ('{ "name": "Bob", "age": 25, "email": "bob@example.com", "active": true }') if attached value and then value.is_object then obj := value.as_object -- Type-safe access with built-in validation if obj.has_key ("name") then print (obj.string_item ("name")) -- Bob end if obj.has_key ("age") then print (obj.integer_item ("age")) -- 25 end if obj.has_key ("active") then print (obj.boolean_item ("active")) -- True end end end

Example 2: Building Complex Structures

local json: SIMPLE_JSON user: SIMPLE_JSON_OBJECT address: SIMPLE_JSON_OBJECT tags: SIMPLE_JSON_ARRAY do create json -- Create nested address address := json.new_object .put_string ("123 Main St", "street") .put_string ("Springfield", "city") .put_string ("62701", "zip") -- Create tags array tags := json.new_array .add_string ("developer") .add_string ("eiffel") .add_string ("json") -- Create main user object user := json.new_object .put_string ("Alice", "name") .put_integer (30, "age") .put_object (address, "address") .put_array (tags, "tags") print (user.to_json_string) end

Example 3: Error Handling with Position Information

local json: SIMPLE_JSON value: detachable SIMPLE_JSON_VALUE do create json value := json.parse ('{ "name": "Test", "invalid": [1, 2, 3,] -- Trailing comma error }') if value = Void and then json.has_errors then across json.last_errors as ic loop print ("Error at line " + ic.line.out) print (", column " + ic.column.out) print (": " + ic.message + "%N") -- Print source context if attached ic.source_line as l_line then print (l_line + "%N") print (ic.position_indicator) end end end end

Example 4: File Parsing with Error Recovery

local json: SIMPLE_JSON value: detachable SIMPLE_JSON_VALUE retry_count: INTEGER do create json from retry_count := 0 until value /= Void or retry_count > 3 loop value := json.parse_file ("config.json") if value = Void then if json.has_errors then print ("Parse error: " + json.errors_as_string + "%N") -- Check if it's a file access error if attached json.first_error as err then if err.message.has_substring ("Cannot read file") then print ("File access error - aborting%N") retry_count := 4 -- Exit loop else -- Syntax error - might be corrupted file print ("Retrying...%N") retry_count := retry_count + 1 end end end end end if attached value then print ("Successfully parsed configuration") end end

Common Use Cases

Configuration Files

Parse application configuration from JSON files:

load_config: SIMPLE_JSON_OBJECT local json: SIMPLE_JSON value: detachable SIMPLE_JSON_VALUE do create json value := json.parse_file ("app_config.json") if attached value and then value.is_object then Result := value.as_object else -- Return default config create Result.make Result.put_string ("localhost", "host") .put_integer (8080, "port") end ensure result_not_void: Result /= Void end

API Response Parsing

Parse JSON responses from web APIs:

parse_api_response (response_text: STRING_32): BOOLEAN local json: SIMPLE_JSON value: detachable SIMPLE_JSON_VALUE obj: SIMPLE_JSON_OBJECT do create json value := json.parse (response_text) if attached value and then value.is_object then obj := value.as_object -- Check for success status if obj.has_key ("status") then Result := obj.string_item ("status").is_case_insensitive_equal ("success") end -- Process data if successful if Result and obj.has_key ("data") then process_response_data (obj.item ("data")) end end end

Data Export

Build JSON for data export:

export_users (users: LIST [USER]): STRING_32 local json: SIMPLE_JSON array: SIMPLE_JSON_ARRAY user_obj: SIMPLE_JSON_OBJECT do create json array := json.new_array across users as ic loop user_obj := json.new_object .put_string (ic.name, "name") .put_string (ic.email, "email") .put_integer (ic.age, "age") array.add_object (user_obj) end Result := array.to_json_string end

Validation Before Processing

Validate JSON structure before expensive operations:

process_if_valid (json_text: STRING_32) local json: SIMPLE_JSON value: detachable SIMPLE_JSON_VALUE do create json -- Quick validation check if json.is_valid_json (json_text) then -- Only parse if valid value := json.parse (json_text) if attached value then perform_expensive_processing (value) end else log_validation_errors (json.errors_as_string) end end

Corner Cases and Special Scenarios

Primitive Values at Top Level

SIMPLE_JSON automatically handles primitive values that standard JSON parsers reject:

local json: SIMPLE_JSON value: detachable SIMPLE_JSON_VALUE do create json -- These work even though they're not valid JSON documents value := json.parse ('"just a string"') -- String value value := json.parse ('42') -- Number value value := json.parse ('true') -- Boolean value value := json.parse ('null') -- Null value -- Parser wraps them in array internally then extracts end
Note: The JSON specification (RFC 8259) requires top-level values to be objects or arrays. SIMPLE_JSON relaxes this requirement for convenience by automatically wrapping primitive values.

Empty Input Handling

local json: SIMPLE_JSON value: detachable SIMPLE_JSON_VALUE do create json -- Empty string triggers precondition violation -- value := json.parse ("") -- PRECONDITION VIOLATION! -- Correct: Check before parsing if not input.is_empty then value := json.parse (input) end end
‚ö†Ô∏è Precondition: Both parse and is_valid_json require non-empty input. Check is_empty before calling these methods.

Whitespace Variations

local json: SIMPLE_JSON value: detachable SIMPLE_JSON_VALUE do create json -- All these are valid and equivalent value := json.parse ('{"a":1}') -- Compact value := json.parse ('{ "a": 1 }') -- Spaced value := json.parse ('{%N "a": 1%N}') -- Newlines value := json.parse ('{%T"a":%T1%T}') -- Tabs value := json.parse (' %N { "a" : 1 } ') -- Mixed end

Error Recovery Patterns

parse_with_fallback (json_text: STRING_32): SIMPLE_JSON_VALUE local json: SIMPLE_JSON value: detachable SIMPLE_JSON_VALUE do create json value := json.parse (json_text) if value = Void then -- Parse failed - return safe default create Result.make (create {JSON_NULL}) -- Log error for debugging log_parse_error (json.errors_as_string) else Result := value end ensure result_not_void: Result /= Void end

Design by Contract

Preconditions

All parsing methods require:

  • not_empty: Input text/file path must not be empty

Postconditions

parse guarantees:

  • errors_cleared_on_success: If Result is not Void, no errors exist

is_valid_json guarantees:

  • valid_implies_no_errors: If returns True, no errors exist

Class Invariants

All error tracking features maintain:

  • last_errors is never Void (may be empty)
  • has_errors is consistent with last_errors.is_empty
  • error_count equals last_errors.count
